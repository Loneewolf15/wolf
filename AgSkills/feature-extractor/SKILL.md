# Feature Extractor

## Overview

This skill helps you analyze a Product Requirement Document (PRD) to find opportunities for "feature packing"—adding high-value, often unexpected features that significantly enhance the user experience and perceived value of the application. It focuses on identifying gaps between a "standard" app and a "premium/packed" app.

## Workflow

### 1. Analyze the PRD

Read the provided PRD and determine:
- **Core Value Proposition**: What is the main problem this app solves?
- **Target Audience**: Who is the primary user?
- **Current Feature Set**: What is already planned?

### 2. Identify "Packed" Opportunities

Look for features that are:
- **High Utility**: Solves a secondary but frequent problem for the user.
- **"Wow" Factor**: Creates a moment of delight or surprise.
- **Synergistic**: Works well with existing features but adds a new dimension.
- **Low Friction/High Reward**: Easy for the user to use, but provides significant value.

*Think about:*
- "If I were the user, what small thing would make me love this app?"
- "What feature do top competitors have that this app is missing?"
- "What feature do top competitors NOT have that would be a game changer?"

### 3. Propose a Feature

Select the single best feature idea and present it clearly.

## Output Format

When proposing a feature, use the following format:

### 🚀 Proposed Feature: [Feature Name]

**Value Proposition:**
[1-2 sentences explaining why this feature is valuable and how it makes the app more "packed".]

**How it Works:**
[Brief description of the user flow or functionality.]

**Technical Feasibility:**
[Brief assessment of difficulty - Low/Medium/High]

**Why it fits:**
[Connection to the specific PRD context.]
